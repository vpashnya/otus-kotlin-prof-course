### Формулировка задания

**Цель:** Оптимизировать производительность кода, используя корутины, чтобы два поиска выполнялись параллельно, а не
последовательно.

**Задача:**

1. **Модифицировать функцию `findNumberInList`:**
    * Преобразуйте её в `suspend`-функцию, добавив ключевое слово `suspend` в сигнатуру.
    * Замените блокирующий вызов `Thread.sleep(2000L)` на неблокирующий аналог из библиотеки корутин — `delay(2000L)`.
      Это позволит освободить поток для других задач во время "ожидания".

2. **Модифицировать тест `easyHw`:**
    * Оберните логику теста в билдер корутин, например, `runBlocking`, чтобы внутри можно было вызывать `suspend`
      -функции.
    * Запустите два вызова `findNumberInList` **параллельно**. Для этого идеально подходит билдер `async`, который
      запускает корутину и возвращает `Deferred` — "обещание" будущего результата.
    * Используйте `await()` для получения результатов от обоих `Deferred`-объектов.

**Ожидаемый результат:**

После выполнения задания общее время работы теста должно сократиться с ~4 секунд до ~2 секунд, так как оба поиска будут
выполняться одновременно.
